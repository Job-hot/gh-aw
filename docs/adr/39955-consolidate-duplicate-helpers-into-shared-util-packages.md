# ADR-39955: Consolidate Duplicate Helpers into Shared Util Packages

**Date**: 2026-06-18
**Status**: Draft
**Deciders**: Unknown (PR #39955 author / reviewers)

---

## Part 1 — Narrative (Human-Friendly)

### Context

Semantic analysis across ~920 non-test Go files surfaced seven categories of duplicate or scattered helper logic: three independent `${{ }}` expression detectors, ad-hoc `[]any → []string` converters, repeated `seen := make(map[string]bool)` dedup loops at 27+ sites, near-identical slice-merge functions, and ~85% shared Node.js parser scaffolding between two CLI parsers. This duplication makes behavior drift likely (one copy gets fixed, others don't) and obscures the canonical way to perform common operations. The codebase already has dedicated utility packages (`pkg/stringutil`, `pkg/sliceutil`, `pkg/typeutil`) and a precedent (`ADR-27198`) for delegating private helpers to exported canonical functions.

### Decision

We will eliminate the identified duplication by consolidating each category into a single canonical location and having former call sites delegate to it. Specifically: expression detection moves to exported `stringutil.HasExpressionMarker`/`ContainsExpression`/`IsExpression` (with `expression_patterns.go` delegating per `ADR-27198`); dedup loops standardize on `map[string]struct{}` and the new `sliceutil.DeduplicateTrimmed`; `[]any → []string` conversion consolidates into `anySliceToStrings`; and the shared Node.js parser scaffolding extracts into `logs_parsing_runner.go` (`jsCoreMock` + `runNodeScript`). Intentionally distinct helpers (e.g. `ShortenCommand` with content-cap semantics) are left in place.

### Alternatives Considered

#### Alternative 1: Leave duplication in place
Keeping each helper local avoids cross-package coupling and a large diff. Rejected because behavioral drift between copies is a recurring real cost and the duplication actively hides the intended canonical pattern from contributors.

#### Alternative 2: Automated codemod / linter to forbid the patterns without manual consolidation
A lint rule banning `map[string]bool` dedup could enforce the convention going forward. Rejected as insufficient on its own — it would flag existing sites without providing the shared destination helpers, and several consolidations (Node.js scaffolding, slice merges) require structural extraction a linter cannot perform. A lint rule remains a viable future complement.

### Consequences

#### Positive
- Single source of truth per helper category — a fix or behavior change applies everywhere at once.
- `map[string]struct{}` dedup communicates set-membership intent and avoids the unused `bool` value.
- New contributors discover the canonical helper instead of copying a local variant.

#### Negative
- More call sites now depend on shared util packages, increasing coupling and the blast radius of a bug in those packages.
- Large mechanical diff touching `pkg/workflow`, `pkg/cli`, and `pkg/parser` raises review burden and rebase/merge-conflict risk for in-flight branches.

#### Neutral
- Cascading signature/field changes (`frontmatterImportsOpts.seen`, `collectLocalIncludeDependenciesRecursive`) follow from the `map[string]struct{}` switch.
- Some helpers (`ShortenCommand`) are deliberately excluded; the consolidation is targeted, not exhaustive.

## Part 2 — Normative Specification (RFC 2119)

> The key words **MUST**, **MUST NOT**, **REQUIRED**, **SHALL**, **SHALL NOT**, **SHOULD**, **SHOULD NOT**, **RECOMMENDED**, **MAY**, and **OPTIONAL** in this section are to be interpreted as described in [RFC 2119](https://www.rfc-editor.org/rfc/rfc2119).

### Helper Consolidation

1. Code that detects GitHub Actions `${{ }}` expressions **MUST** use the exported `pkg/stringutil` helpers rather than reimplementing the `strings.Contains(v, "${{") && strings.Contains(v, "}}")` check inline.
2. New string-set deduplication **MUST NOT** introduce `map[string]bool`; it **MUST** use `map[string]struct{}` or an appropriate `pkg/sliceutil` helper.
3. Trim-and-dedup-preserving-order over a `[]string` **SHOULD** use `sliceutil.DeduplicateTrimmed` rather than a hand-written loop.
4. Helpers with semantics genuinely distinct from an existing canonical function (e.g. content-cap truncation) **MAY** remain separate, but the divergence **SHOULD** be documented in a comment.

### Conformance

An implementation is conformant with this ADR if it satisfies all **MUST** and **MUST NOT** requirements above. Failure to meet any **MUST** or **MUST NOT** requirement constitutes non-conformance.
